# Infrastructure Engineering PR Checklist

One page. Pull this up during infra/platform reviews. Every item is a `must`-severity rule from the guide — copy/paste lines into PRs, tick what holds, link the chapter for depth.

## 1. Foundations

- [ ] Define all infrastructure as code in version control; no console click-ops. → [ch. 01](docs/01-foundations.md)
- [ ] Write declarative definitions, not imperative scripts. → [ch. 01](docs/01-foundations.md)
- [ ] Make every change idempotent and re-runnable. → [ch. 01](docs/01-foundations.md)
- [ ] Keep code, state, and secrets in separate access-controlled stores. → [ch. 01](docs/01-foundations.md)
- [ ] Detect drift continuously, not when something breaks. → [ch. 01](docs/01-foundations.md)
- [ ] Model infra systems as control-loop reconcilers. → [ch. 01](docs/01-foundations.md)
- [ ] Only call it GitOps if it satisfies all four OpenGitOps principles (declarative, versioned, pulled, reconciled). → [ch. 01](docs/01-foundations.md)
- [ ] Produce environments from the same code with different parameters; never copy-paste. → [ch. 01](docs/01-foundations.md)
- [ ] Size stacks so a worst-case apply only destroys what you can afford to lose. → [ch. 01](docs/01-foundations.md)
- [ ] The team that builds a service operates it; no separate ops group owning prod. → [ch. 01](docs/01-foundations.md)
- [ ] Treat operability as a first-class requirement of every change, not a follow-up ticket. → [ch. 01](docs/01-foundations.md)

## 2. IaC tooling

- [ ] Use one IaC tool per repo and one CLI per state file; never mix Terraform and OpenTofu against the same state. → [ch. 02](docs/02-iac-tooling.md)
- [ ] Keep modules single-purpose; compose only in the root module. → [ch. 02](docs/02-iac-tooling.md)
- [ ] Use remote state with locking on every environment; no local state in CI. → [ch. 02](docs/02-iac-tooling.md)
- [ ] Encrypt state at rest, restrict bucket access to the apply role, version objects, and log access. → [ch. 02](docs/02-iac-tooling.md)
- [ ] Keep one state file per blast-radius unit (per-env, per-lifecycle). → [ch. 02](docs/02-iac-tooling.md)
- [ ] Run on every PR at least one static gate (`fmt -check`, `validate`, lint) and one policy/security scanner (Checkov, KICS, Trivy, cdk-nag, psrule). → [ch. 02](docs/02-iac-tooling.md)
- [ ] Never commit plaintext secrets in `.tf`, `.tfvars`, Bicep params, Pulumi config, or CDK source — including dev values. → [ch. 02](docs/02-iac-tooling.md)
- [ ] Treat `plan` output as secret-bearing; never paste unredacted into PR comments or public CI logs. → [ch. 02](docs/02-iac-tooling.md)
- [ ] Commit the provider lock file and pin provider versions with an upper bound. → [ch. 02](docs/02-iac-tooling.md)
- [ ] Attach a `plan` artifact to every PR; reviewers approve the plan, not just the diff. → [ch. 02](docs/02-iac-tooling.md)

## 3. CI/CD

- [ ] Define pipelines as code next to the application; review in PRs, no UI-only config. → [ch. 03](docs/03-ci-cd.md)
- [ ] Deploy to Kubernetes pull-based via Argo CD or Flux; no `kubectl apply` from CI. → [ch. 03](docs/03-ci-cd.md)
- [ ] Reference container images by digest (`@sha256:…`) in the env repo, never by mutable tag. → [ch. 03](docs/03-ci-cd.md)
- [ ] Ship every release artifact with a signed SLSA provenance attestation at the level defined in ch06 §9 (currently L3 for production artifacts). → [ch. 03](docs/03-ci-cd.md), [ch. 06 §9](docs/06-security-supply-chain.md)
- [ ] Generate an SBOM at build time for every image and release binary; store as an OCI referrer. → [ch. 03](docs/03-ci-cd.md)
- [ ] Sign every container and release artifact with cosign keyless using the CI's OIDC identity. → [ch. 03](docs/03-ci-cd.md)
- [ ] Verify signatures at admission in the cluster (policy-controller, Kyverno, Ratify, Connaisseur), not only in CI. → [ch. 03](docs/03-ci-cd.md)
- [ ] Pin every third-party action to a 40-char commit SHA; tags are mutable. → [ch. 03](docs/03-ci-cd.md)
- [ ] Federate cloud auth via OIDC scoped to repo + workflow + branch/environment; no long-lived cloud creds in CI. → [ch. 03](docs/03-ci-cd.md)
- [ ] Declare an explicit top-level `permissions:` block defaulting to `contents: read`; elevate per job. → [ch. 03](docs/03-ci-cd.md)
- [ ] Protect the default branch with Repository Rulesets: required reviews, required checks (build/test/SBOM/sign/Scorecard/pin-lint), no force-push, no admin bypass. → [ch. 03](docs/03-ci-cd.md)
- [ ] Provide an automated, tested rollback path for every prod deploy (previous digest, blue/green flip, `argo rollouts undo`). → [ch. 03](docs/03-ci-cd.md)
- [ ] Instrument the DORA metrics defined in ch11 §14 and surface them to the owning team. → [ch. 03](docs/03-ci-cd.md), [ch. 11 §14](docs/11-platform-engineering.md)
- [ ] Use ephemeral runners; persist nothing between jobs except an explicit scoped cache. → [ch. 03](docs/03-ci-cd.md)
- [ ] Scope caches per-branch keyed by lockfile hash; never accept untrusted-PR caches into privileged workflows. → [ch. 03](docs/03-ci-cd.md)

## 4. Containers & Kubernetes

- [ ] Use a distroless or Wolfi/Chainguard runtime image with no shell or package manager. → [ch. 04](docs/04-containers-k8s.md)
- [ ] Multi-stage build; the runtime stage contains no compiler or build tools. → [ch. 04](docs/04-containers-k8s.md)
- [ ] Run as non-root with a read-only root filesystem. → [ch. 04](docs/04-containers-k8s.md)
- [ ] Scan images on push and on a daily schedule; block deploy on critical CVEs. → [ch. 04](docs/04-containers-k8s.md)
- [ ] Set CPU and memory requests > 0 on every container. → [ch. 04](docs/04-containers-k8s.md)
- [ ] Set a memory limit; rely on OOM-kill rather than node exhaustion. → [ch. 04](docs/04-containers-k8s.md)
- [ ] Keep readiness probes free of external dependencies (DB, downstream services). → [ch. 04](docs/04-containers-k8s.md)
- [ ] Handle SIGTERM gracefully and pair with a `preStop` sleep to win the endpoints race. → [ch. 04](docs/04-containers-k8s.md)
- [ ] Ship a PDB and topology spread constraints with every multi-replica Deployment. → [ch. 04](docs/04-containers-k8s.md)
- [ ] Enforce PodSecurity `restricted` at the namespace level on all workload namespaces. → [ch. 04](docs/04-containers-k8s.md)
- [ ] Drop ALL capabilities, set seccomp `RuntimeDefault`, disallow privilege escalation. → [ch. 04](docs/04-containers-k8s.md)
- [ ] Disallow `privileged`, `hostNetwork`, `hostPID`, and `hostPath` in application workloads. → [ch. 04](docs/04-containers-k8s.md)
- [ ] Apply a default-deny NetworkPolicy in every namespace plus explicit allow-lists. → [ch. 04](docs/04-containers-k8s.md)
- [ ] Standardize on containerd (or CRI-O) on cluster nodes; the Docker daemon is obsolete in K8s. → [ch. 04](docs/04-containers-k8s.md)
- [ ] Use managed Kubernetes (AKS/EKS/GKE) unless a regulatory or hardware reason forbids it. → [ch. 04](docs/04-containers-k8s.md)

## 5. Observability

- [ ] Treat observability as the ability to ask new questions of a running system without shipping code. → [ch. 05](docs/05-observability.md)
- [ ] Instrument around wide, high-cardinality structured events; logs/metrics/traces are projections. → [ch. 05](docs/05-observability.md)
- [ ] Use OpenTelemetry SDKs and OTLP for all new instrumentation; no vendor SDKs in app code. → [ch. 05](docs/05-observability.md)
- [ ] Run an OpenTelemetry Collector between apps and backends for batching, redaction, sampling, and backend swap. → [ch. 05](docs/05-observability.md)
- [ ] Propagate W3C Trace Context (`traceparent`/`tracestate`) on every hop; no custom trace headers. → [ch. 05](docs/05-observability.md)
- [ ] Trace external requests and async boundaries (HTTP, gRPC, queues, DB drivers); a trace stopping at the service edge is theatre. → [ch. 05](docs/05-observability.md)
- [ ] Use RED per endpoint and USE per resource; pair with the four golden signals at the edge. → [ch. 05](docs/05-observability.md)
- [ ] Forbid unbounded labels (user_id, request_id, raw URL paths, full error strings) on Prometheus metrics. → [ch. 05](docs/05-observability.md)
- [ ] Emit logs as JSON, one event per line, via a structured logger; no printf logging in prod. → [ch. 05](docs/05-observability.md)
- [ ] Adopt one log schema across the estate — OpenTelemetry Logs (preferred) or ECS. → [ch. 05](docs/05-observability.md)
- [ ] Carry `trace_id` and `span_id` on every log line via the OTel logs bridge. → [ch. 05](docs/05-observability.md)
- [ ] Use tail-based sampling in the Collector: keep 100% errors, 100% slow traces, plus a probabilistic baseline. → [ch. 05](docs/05-observability.md)
- [ ] Sample at trace granularity, not span granularity. → [ch. 05](docs/05-observability.md)
- [ ] Page only on user-visible SLO burn, not on CPU/memory. → [ch. 05](docs/05-observability.md)
- [ ] Use multi-window, multi-burn-rate SLO alerts (fast 5m+1h ≥14.4, slow 30m+6h ≥6). → [ch. 05](docs/05-observability.md)
- [ ] Codify SLOs (sloth/pyrra/OpenSLO) so docs and Prometheus rules cannot drift. → [ch. 05](docs/05-observability.md)

## 6. Security & supply chain

- [ ] Maintain an in-repo threat model (STRIDE; LINDDUN if PII) for every prod system; review on architectural change. → [ch. 06](docs/06-security-supply-chain.md)
- [ ] Store secrets in a managed store (Vault / cloud KMS / Secret Manager) or SOPS-encrypted in git — never plaintext in VCS, images, or env files. → [ch. 06](docs/06-security-supply-chain.md)
- [ ] Enforce secret rotation in the store, not on a calendar. → [ch. 06](docs/06-security-supply-chain.md)
- [ ] Use IRSA / EKS Pod Identity / GKE WI / AKS WI / OIDC federation; no long-lived static cloud creds in CI or workloads. → [ch. 06](docs/06-security-supply-chain.md)
- [ ] Scope OIDC trust policies to repo + branch + environment + (for PR jobs) workflow. → [ch. 06](docs/06-security-supply-chain.md)
- [ ] Enforce platform invariants with an admission controller (Kyverno or Gatekeeper), not wiki pages. → [ch. 06](docs/06-security-supply-chain.md)
- [ ] Run containers as non-root, read-only rootfs, no privileged, drop ALL caps, seccomp `RuntimeDefault`. → [ch. 06](docs/06-security-supply-chain.md)
- [ ] Ship an SBOM (SPDX 2.3+ or CycloneDX 1.5+) generated at build time with every published artifact. → [ch. 06](docs/06-security-supply-chain.md)
- [ ] Attach SBOMs as signed cosign attestations and ingest them into a vuln DB; don't ship SBOMs you don't read. → [ch. 06](docs/06-security-supply-chain.md)
- [ ] Scan in three places: PR diff, post-build full image, and continuously in the registry. → [ch. 06](docs/06-security-supply-chain.md)
- [ ] Target SLSA Build L3 for production artifacts with hosted, hardened, non-falsifiable provenance. → [ch. 06](docs/06-security-supply-chain.md)
- [ ] Sign every container, chart, and release artifact with cosign; verify at admission. → [ch. 06](docs/06-security-supply-chain.md)
- [ ] Pin verification policies to the *signer identity*, not just "has a signature". → [ch. 06](docs/06-security-supply-chain.md)
- [ ] Adopt BeyondCorp posture: workloads on private subnets, no public IPs by default, identity at the application layer. → [ch. 06](docs/06-security-supply-chain.md)
- [ ] Apply default-deny NetworkPolicy in every app namespace per ch04 §8 (canonical rule). → [ch. 06](docs/06-security-supply-chain.md), [ch. 04 §8](docs/04-containers-k8s.md)
- [ ] Require hardware-backed WebAuthn for all human production access; no SMS/TOTP-only paths. → [ch. 06](docs/06-security-supply-chain.md)
- [ ] Publish a dashboarded patch SLA by severity and exposure. → [ch. 06](docs/06-security-supply-chain.md)
- [ ] Enable CloudTrail / Activity Log / Cloud Audit Logs in every region/subscription/project, centralised to a write-once store. → [ch. 06](docs/06-security-supply-chain.md)
- [ ] Run a runtime threat-detection agent (Falco / Tetragon) on every prod node with tuned, routed rules. → [ch. 06](docs/06-security-supply-chain.md)
- [ ] Own audit logging, encryption at rest, access reviews, change-management trail, and tested backups as infra responsibilities. → [ch. 06](docs/06-security-supply-chain.md)
- [ ] Read xz (CVE-2024-3094), SUNBURST, and Codecov postmortems; design reviews must answer "what would have caught this here?". → [ch. 06](docs/06-security-supply-chain.md)

## 7. Networking

- [ ] Know the AWS/Azure/GCP primitive mapping cold; subnet scope and VPC scope drive every topology decision. → [ch. 07](docs/07-networking.md)
- [ ] Allocate /16 per VPC with /24 subnets from a written central IPAM plan; never overlap CIDRs across anything that might one day connect. → [ch. 07](docs/07-networking.md)
- [ ] Adopt a Landing Zone model (CAF / AWS Control Tower) from day one; central connectivity account owns the hub. → [ch. 07](docs/07-networking.md)
- [ ] Use hub-and-spoke via Transit Gateway / Virtual WAN / NCC beyond ~5 VPCs; never grow a mesh-peering topology. → [ch. 07](docs/07-networking.md)
- [ ] Terminate every supported PaaS over PrivateLink / Private Endpoint / PSC; disable the public endpoint. → [ch. 07](docs/07-networking.md)
- [ ] Run split-horizon DNS with private zones centralized in the hub; spokes use conditional forwarding. → [ch. 07](docs/07-networking.md)
- [ ] Default to no public IPs on workloads; only edge LBs and NAT/firewall have them. → [ch. 07](docs/07-networking.md)
- [ ] Egress through a centralized hub firewall with FQDN-allowlist policy in production; NAT-only is dev-grade. → [ch. 07](docs/07-networking.md)
- [ ] Apply BCP 38 egress filtering at every edge you control. → [ch. 07](docs/07-networking.md)
- [ ] Authenticate and authorize every request and use mTLS for service-to-service; the network is not the perimeter. → [ch. 07](docs/07-networking.md)
- [ ] Replace user VPN with an identity-aware proxy (Cloudflare Access / Tailscale / IAP / Pomerium). → [ch. 07](docs/07-networking.md)
- [ ] Pick L4 vs L7 deliberately; L4 for raw TCP/UDP and source-IP preservation, L7 for HTTP/WAF/gRPC. → [ch. 07](docs/07-networking.md)
- [ ] Keep health checks cheap, deterministic, and unauthenticated; never call the database from `/healthz`. → [ch. 07](docs/07-networking.md)
- [ ] Enable connection draining / deregistration delay on every target group (30–60s default; longer for websockets). → [ch. 07](docs/07-networking.md)
- [ ] Terminate TLS at the edge AND re-encrypt to the workload (or mesh sidecar); end-to-end is the zero-trust default. → [ch. 07](docs/07-networking.md)
- [ ] Automate cert issuance and renewal with ACME; in K8s use cert-manager with Issuer/ClusterIssuer. → [ch. 07](docs/07-networking.md)
- [ ] Use Direct Connect / ExpressRoute / Cloud Interconnect for site-to-site; pair with an IPsec VPN failover. → [ch. 07](docs/07-networking.md)
- [ ] Use BGP with BFD on hybrid links; static routes don't fail over. → [ch. 07](docs/07-networking.md)

## 8. Data & state

- [ ] Default to a managed service for Postgres, MySQL, Redis, Kafka, object storage, and search; require a written exception to self-host. → [ch. 08](docs/08-data-state.md)
- [ ] Apply 3-2-1 backups: live DB + same-region snapshots + cross-account, cross-region, object-locked off-site copy written by a separate identity. → [ch. 08](docs/08-data-state.md)
- [ ] Enable PITR on every system of record with retention ≥14 days; alert when oldest restorable timestamp regresses. → [ch. 08](docs/08-data-state.md)
- [ ] Encrypt backups with a CMK in a separate trust domain from the workload identity that runs the database. → [ch. 08](docs/08-data-state.md)
- [ ] Run a timed, end-to-end DR drill at least quarterly (monthly for tier-0); record restore time as an SLI. → [ch. 08 §3](docs/08-data-state.md)
- [ ] Dashboard "time since last successful restore" for every tier-0/1 store and page when it exceeds policy. → [ch. 08](docs/08-data-state.md)
- [ ] Derive RPO and RTO from the customer-facing SLO; reject any RTO larger than the monthly error budget. → [ch. 08](docs/08-data-state.md)
- [ ] Validate RPO/RTO with quarterly game-day failovers; declared values without test evidence are invalid. → [ch. 08](docs/08-data-state.md)
- [ ] Automate failover (managed failover, Patroni, orchestrator, MySQL InnoDB Cluster) and chaos-test it; no manual-DBA failover paths. → [ch. 08](docs/08-data-state.md)
- [ ] Document the shard key in an ADR; high-cardinality, present on every hot query, aligned with tenancy. → [ch. 08](docs/08-data-state.md)
- [ ] One database per service; no shared schemas across services with different release cadences. → [ch. 08](docs/08-data-state.md)
- [ ] Treat caches as ephemeral; never store data that cannot be regenerated from the system of record. → [ch. 08](docs/08-data-state.md)
- [ ] Make every consumer idempotent; do not rely on broker "exactly-once" semantics. → [ch. 08](docs/08-data-state.md)
- [ ] Block public access at the account level for every object-storage account. → [ch. 08](docs/08-data-state.md)
- [ ] Enable versioning + lifecycle expiry for noncurrent versions; enable Object Lock on backup/audit/compliance buckets. → [ch. 08](docs/08-data-state.md)
- [ ] Pin tenant data, backups, logs, and traces to permitted regions; record the home region at provisioning. → [ch. 08](docs/08-data-state.md)
- [ ] Use per-residency-zone CMKs; key material does not cross borders. → [ch. 08](docs/08-data-state.md)
- [ ] Document erasure propagation across every store (warehouse, search, cache, broker, backup); event-sourced systems need a crypto-shredding plan. → [ch. 08](docs/08-data-state.md)
- [ ] Use expand/contract (parallel change) for migrations; never ship one that requires app and schema to deploy atomically. → [ch. 08](docs/08-data-state.md)
- [ ] Apply migrations through a versioned tool (Atlas / Flyway / Liquibase / Sqitch); reject ad-hoc DDL by humans in prod. → [ch. 08](docs/08-data-state.md)
- [ ] Encrypt at rest with a CMK for any regulated or production data; verify key rotation is automated. → [ch. 08](docs/08-data-state.md)
- [ ] Require TLS 1.2+ (1.3 preferred) on every database connection, including intra-VPC. → [ch. 08](docs/08-data-state.md)
- [ ] If self-hosting on K8s, push backups to object storage outside the cluster, written by an identity the cluster cannot revoke. → [ch. 08](docs/08-data-state.md)

## 9. Reliability

- [ ] Ship every user-facing service with at least one SLI/SLO and a written error-budget policy. → [ch. 09](docs/09-reliability.md)
- [ ] Define SLIs as ratios of good/valid events on user journeys, not infra metrics. → [ch. 09](docs/09-reliability.md)
- [ ] Page humans only on multi-window, multi-burn-rate SLO alerts; everything else is a ticket. → [ch. 09](docs/09-reliability.md)
- [ ] Link a runbook in every paging alert payload. → [ch. 09](docs/09-reliability.md)
- [ ] Adopt explicit ICS roles (IC, Comms, Ops) with channel separation between working and broadcast. → [ch. 09](docs/09-reliability.md)
- [ ] Make blameless postmortems mandatory for SEV1/SEV2 and any near-miss touching auth/data/budget exhaustion; publish within 5 business days. → [ch. 09](docs/09-reliability.md)
- [ ] Give every action item an owner, a ticket, and a due date, tagged prevent vs mitigate. → [ch. 09](docs/09-reliability.md)
- [ ] Write runbooks as version-controlled decision trees; gameday-test at least quarterly. → [ch. 09](docs/09-reliability.md)
- [ ] Compensate on-call and run a rotation with ≥6 humans (or follow-the-sun across regions). → [ch. 09](docs/09-reliability.md)
- [ ] Cap toil at 50% of SRE/platform time; measure and publish quarterly. → [ch. 09](docs/09-reliability.md)
- [ ] Set an explicit timeout on every network call, shorter than the caller's; no infinite waits. → [ch. 09](docs/09-reliability.md)
- [ ] Retry with exponential backoff, full jitter, and a per-client retry budget. → [ch. 09](docs/09-reliability.md)
- [ ] Carry an idempotency key on every retried mutation. → [ch. 09](docs/09-reliability.md)
- [ ] Plan to N+2 with steady-state utilization ≤60–70% and a forecast horizon ≥ slowest-resource lead time. → [ch. 09](docs/09-reliability.md)
- [ ] Practice DR: full region drill ≥annually for tier-1, partial quarterly (ch08 §3 timed end-to-end DR drill); plus a weekly *automated* restore verification loop owned by ch09 §12. → [ch. 09 §12](docs/09-reliability.md), [ch. 08 §3](docs/08-data-state.md)
- [ ] Set RTO/RPO per service from the business and document them next to actual achieved values. → [ch. 09](docs/09-reliability.md)

## 10. FinOps

- [ ] Adopt the FinOps Foundation framework explicitly: Inform → Optimize → Operate, run continuously. → [ch. 10](docs/10-finops.md)
- [ ] Treat FinOps as a cross-functional practice (eng + finance + product), not a finance reporting function. → [ch. 10](docs/10-finops.md)
- [ ] Require new-workload proposals to state the assumed pricing model (on-demand / RI / SP / spot) and break-even horizon. → [ch. 10](docs/10-finops.md)
- [ ] Enforce required tags on every cloud resource in IaC: cost-center, service, env, owner, data-classification. → [ch. 10](docs/10-finops.md)
- [ ] Standardize cross-cloud cost data on the FOCUS specification; do not build bespoke ETL from raw billing exports. → [ch. 10](docs/10-finops.md)
- [ ] Set a budget on every account/subscription/project with at least 50/80/100% thresholds and a documented action per threshold. → [ch. 10](docs/10-finops.md)
- [ ] Implement a non-prod kill-switch: at 100% budget, automatically scale-to-zero or stop dev/test environments. → [ch. 10](docs/10-finops.md)
- [ ] Rightsize from observed p95/p99 utilization with documented headroom (default p99 + 30%); never from synthetic benchmarks. → [ch. 10](docs/10-finops.md)
- [ ] Measure cold-start cost in prod p99 latency before adopting FaaS; document the trade-off in the service README. → [ch. 10](docs/10-finops.md)
- [ ] Use the dual-pool pattern in production K8s: an on-demand floor for system + stateful, spot bulk for stateless. → [ch. 10](docs/10-finops.md)
- [ ] Diversify spot pools across ≥3 instance families/sizes and ≥3 AZs. → [ch. 10](docs/10-finops.md)
- [ ] Put a CDN in front of any object/blob storage that serves the public internet. → [ch. 10](docs/10-finops.md)
- [ ] Define lifecycle policies as IaC for every bucket/container with hot → warm → cold → archive → delete transitions. → [ch. 10](docs/10-finops.md)
- [ ] Account for retrieval and request costs when choosing a storage tier; deep-archive can cost more on access than hot storage. → [ch. 10](docs/10-finops.md)
- [ ] Make unit cost ($/order, $/active user, $/inference) the headline engineering metric, not the absolute monthly bill. → [ch. 10](docs/10-finops.md)
- [ ] Enable cloud-native anomaly detection at the org/management group level. → [ch. 10](docs/10-finops.md)
- [ ] Run a laddered commitment strategy (staggered 1y/3y so commitments expire continuously); never reach 100% coverage. → [ch. 10](docs/10-finops.md)

## 11. Platform engineering

- [ ] Build opinionated, self-service products that reduce stream-aligned teams' cognitive load — not DevOps-renamed, not SRE-for-everyone. → [ch. 11](docs/11-platform-engineering.md)
- [ ] Keep the platform the thinnest viable layer over the underlying primitives; never reinvent them. → [ch. 11](docs/11-platform-engineering.md)
- [ ] Run the platform as a product: named internal customers, public roadmap, SLOs to consumers, deprecation policy, feedback loop. → [ch. 11](docs/11-platform-engineering.md)
- [ ] Validate demand before building — no platform feature ships without an identified consumer team that has agreed to adopt it. → [ch. 11](docs/11-platform-engineering.md)
- [ ] Provide opt-in paved roads, not mandatory walled gardens. → [ch. 11](docs/11-platform-engineering.md)
- [ ] Build the IDP as the integrated product surface; a portal alone is not a platform. → [ch. 11](docs/11-platform-engineering.md)
- [ ] Expose the platform as an API first; the UI is one client. CI, scripting, and IDEs must be able to use it. → [ch. 11](docs/11-platform-engineering.md)
- [ ] Make self-service mean "consumer declares intent in their own repo and a controller reconciles it" — not "raise a ticket". → [ch. 11](docs/11-platform-engineering.md)
- [ ] Pitch abstractions at configuration-as-data over a small declarative spec; raw IaC is too low, full PaaS is too high. → [ch. 11](docs/11-platform-engineering.md)
- [ ] Organise infra work around the four Team Topologies team types and three interaction modes; do not invent a fifth. → [ch. 11](docs/11-platform-engineering.md)
- [ ] Use the Inverse Conway Maneuver: shape the org so the desired architecture is the easy one to build. → [ch. 11](docs/11-platform-engineering.md)
- [ ] Move shared ops concerns to X-as-a-Service so stream-aligned teams' intrinsic cognitive load fits human limits. → [ch. 11](docs/11-platform-engineering.md)
- [ ] Measure platform success by consumer-team DORA metrics and platform-feature adoption rate; not tickets closed. → [ch. 11](docs/11-platform-engineering.md)
- [ ] Don't invest in a platform team before complexity justifies it (~30+ engineers or ~10+ independent services). → [ch. 11](docs/11-platform-engineering.md)
- [ ] Treat documentation as a first-class platform feature: colocated with code, searchable, versioned. ADRs capture *why*, not just *what*. → [ch. 11](docs/11-platform-engineering.md)
